### 3.1 Bug timezone UTC (`toISOString().split('T')[0]`) — 🟡 ALTO

Patrón que retorna la fecha en UTC en lugar de la fecha local del navegador. Monterrey (CST UTC-6) produce fecha adelantada 1 día después de las 18:00 local. **9 ocurrencias**:

- `operaciones/config/index.html:902`
- `operaciones/dashboard/js/dashboard.js:144` — dashboard muestra "sin técnicos" tras 18:00 CST
- `operaciones/incidencias/empleado/index.html:233, 357`
- `operaciones/incidencias/supervisor/index.html:205, 368, 441`
- `shared/auth-suite.js:84` — validación de `vigencia` de usuarios con fecha UTC

**Nota**: PR #4 ya arregló las 2 ocurrencias críticas del kiosk (`kiosk.js:1376, 1518`) con el helper `fechaLocalISO()` — `operaciones/kiosk/js/kiosk.js:38–46` documenta el problema y el workaround. Las 9 restantes aún usan el patrón buggy. **Solución pendiente**: extraer `fechaLocalISO()` a `shared/date-utils.js` y reemplazar los 9 call sites.

### 3.2 Comparaciones `==` en contextos críticos — ✅ OK

**6 ocurrencias totales**, todas son `== null` (captura `null` + `undefined` en un solo check) — patrón idiomático seguro. **No hay bugs introducidos por coerción de tipos.** No requiere acción.

### 3.3 GitHub PAT en localStorage (`ops_github_token`) — 🟡 ALTO

**19 referencias** totales (lectura + escritura), dual-write a `ops_github_token` + `key_github_token` sin cifrado.

**Lecturas críticas**:
- `operaciones/config/index.html:865, 1099`
- `operaciones/incidencias/js/incidencias.js:145, 181`
- `shared/auth-suite.js:25` — `loadUsers()` usa token para GitHub API autenticada

**Riesgo**: plaintext en localStorage → expuesto a XSS, devtools, malware local. **Mitigación propuesta**: proxy n8n para gestión de users (pendiente #1 en SECCIÓN 8) para eliminar la necesidad del PAT en el cliente.

### 4.6 Candado contra entrada duplicada

**Lógica entera en backend.** Frontend solo reacciona a la respuesta.

`operaciones/kiosk/js/kiosk.js:743–748`:
```js
var result = await window.OdooKiosk.registrarCheckin(payload);
var r = Array.isArray(result) ? result[0] : result;
if(r && r.accion_valida === false){
  var errMsg = r.error_msg || 'Error desconocido';
  mostrarErrorCandado(errMsg);   // ← Modal con mensaje del backend
  return;
}
```

`mostrarErrorCandado()` en `kiosk.js:1474–1505` parsea prefijos del error:
- `ZONA_GRIS:<hrs>` → 🟡 "Llevas X hrs desde tu última entrada. ¿Olvidaste checar salida?"
- `ERROR_CRITICO:<hrs>` → 🔴 "Tienes una entrada sin salida de hace X horas. Debes resolver esto primero."
- `Ya tienes` → ℹ️ (típico de duplicado de entrada)

**Source of truth**: `/webhook/kiosk/estado-empleado` (kiosk.js:1112–1121) consulta a Odoo en cada `mostrarEstadoEmpleado(empleado)` — sin cache local, fresh siempre. Devuelve:
- `estado_actual`: `sin_registro` | `activo` | `zona_gris` | `error_critico`
- `registro_abierto`: `{ check_in, id, ... }` o `null`
- `horas_hoy`, `horas_restantes`, `horas_transcurridas`

**Implicación**: si un usuario malicioso intenta entrada duplicada, el frontend la envía igual, el backend la rechaza, y el frontend muestra el modal. **No hay pre-check client-side**. Bandwidth desperdiciado pero no hay riesgo de duplicar attendances en Odoo.
