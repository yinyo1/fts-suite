🌐 Conexiones

URLs base y credenciales de los servicios externos. Se escriben en doble prefijo (ops_* + sin prefijo) para compatibilidad con todos los módulos.

n8n

Odoo
Se sincroniza también como key_odoo_api
Modo

🔑 API Keys

Claves de servicios IA y GitHub. Todas las apps de la suite leen de estas keys estandarizadas. Las keys se cifran con AES-GCM al hacer Sync → GitHub.

ℹ️ Las API Keys se sincronizan a shared/ops-config.json (cifrado) al usar la sección Sync.
Modelos de IA
Dual-write: key_claude + fts_api_key (compat con análisis-estructural)
GitHub
Dual-write: ops_github_token + key_github_token. Necesario para Sync de geolocations y config cifrada.

🔒 Seguridad Industrial

Configuración del módulo Seguridad y IPERC.

ℹ️ El módulo IPERC no usa localStorage propio. Su estado vive en sessionStorage.fts_iperc_session (efímero, se limpia al cerrar el browser).
Configuración requerida

IPERC consume las API Keys globales para análisis de riesgos asistido por IA. Verifica que tengas configuradas en la sección 🔑 API Keys:

  • Claude API Key — análisis principal (Sonnet 4.6)
  • Groq API Key — clasificación rápida de actividades
  • OpenRouter API Key — fallback / modelos alternativos
→ Ir a API Keys ↗ Abrir IPERC

🏗️ Ingeniería

Configuración del módulo de Análisis Estructural con IA.

ℹ️ La Claude API Key se gestiona en la sección 🔑 API Keys. Config Master hace dual-write a fts_api_key automáticamente para que análisis-estructural la lea sin cambios.
System Prompt del Ingeniero IA

Prompt maestro que define el rol de Claude al generar memorias de cálculo. Persistido en fts_system_prompt y leído por análisis-estructural al iniciar.

Dejar vacío para usar el prompt default embebido en prompts/extract.js
